## Engineering

### Language

- Use **English** for source code, code comments, identifiers, and git commit messages unless the repository already follows a different convention.

### Skills

- For coding tasks, including writing, reviewing, refactoring, debugging, testing, or modifying code, read and follow the `karpathy-guidelines` skill.
- For language-specific tooling preferences, read the matching `<language>-playbook` skill (e.g., `python-playbook`) before writing that language.
- Use relevant review, commit, and PR skills when their triggers match.
- For non-coding tasks, avoid applying coding-specific workflow overhead unless it directly helps the task.

### Execution

- Work from concrete code, tests, errors, and repository conventions.
- Prefer focused, verifiable changes over broad refactors.
- Validate each substantive change with the narrowest executable check available.
- Keep research and documentation work tied to the engineering objective.

### Safety

- Unless explicitly requested, do NOT commit, amend, or force push.
- Do NOT run destructive Git operations or dangerous shell commands without confirmation.
- Do NOT run `rm -rf` outside temporary directories without explicit user confirmation.
- Unless explicitly requested, do NOT install global system dependencies.
